That being said, I also represent the majority of desktop users, so this review should apply to most people.<br><br>Remember when Windows Snap was a thing you could do in Windows Vista? I don't blame you, I don't use this feature as much either, but it was there and it lets you precisely snap windows on either side, then later on, top to bottom, this feature that was a staple of many Linux environments (along with NTFS symbolic links) was a welcome and well-integrated addition in Microsoft Windows.<br><br>Recently, in Windows 10, snapping to corners was another feature, and if two windows are snapped together, then you can scale them both at once. There are some issues with some programs (Skype, for instance likes to always screw up when trying to scale a separated friend list with conversations), but it pretty much does what it needs to in most applicatinons.<br><br>AquaSnap also apes some features from different open-source desktops and window managers, but those are available only for open-source operating systems. Honestly, what's already there in Windows is good enough for most users, so it makes you wonder: do you really need to spend fifteen dollars just to improve upon these features, even if only a little bit? Seems quite useless, until you look at the other features this software provides;<br><br><b>AquaSnap<\/b><br>This feature doesn't just improve upon the idea of snapping WIndows, it also adds other small extras like having a snap indicator to tell you how exactly the window will be placed, as well indicators that show a little graphical representation of what will happen. A little redundant, but if you like to customize the icon overlays are capable of being modified rather easily to suit your theme.<br><br>You can even have child windows (the windows inside of windows) use specific settings, if you wanted to take it that far, and limit yourself to snap options that make sense to you, leaving the rest if you don't want those specific features; The default snap functions in Windows doesn't allow for this degree of customization.<br><br><b>AquaStretch<\/b><br>This is more at-home for users of some open-source desktops where maximize horizontally and maximize vertically are options. This really makes little sense for parent windows, but for children windows this feature might come in handy, allowing for a child window to take up space in one direction whule leaving space for others, to be stretched in the same way. While it's nice, as a regular desktop user with a single display it seems fairly useless to me.<br><br><b>AquaMagnet<\/b><br>This feature is very, very similar to window snapping in Compiz, an open-source desktop compositor, with some extra bells and whistles; You can snap <i>inside<\/i> other windows, and snap at corners on top of snapping at the screen edge and outside other windows, and these features can be disabled at-will to make the snap feature work as much as you need it to. This is surprisingly useful if you find yourself needing to snap two windows of dispariate sizes, or two windows that open as the same size, but not taking up all available screen real estate.<br><br><b>AquaGlue<\/b><br>This is truly an interesting feature; You can not just move multiple windows, but also resize multiple windows so you can scale that one window while also not having the other windows nearby overlap. This does a neat trick Windows 10 doesn't do out of the box; You can scale more than two windows at once, independent of what application they are, and independent of where they are at on the screen. so long their edges or corners are touching.<br><br>A practical use of this is having four windows open at once, snapped at the corners, all of them being resized at once. Not just that, you can move entire sets of windows as if they were one big multi-document application, so if your screen is of the same size, then AquaMagnet's edge snapping allows you to just be near a top corner to snap all four windows at once.<br><br><b>AquaShake<\/b><br>By default, the AquaShake feature allows you to put a window always on top by &quot;Shaking&quot; it (with rapid mouse movements), and also make the window semi transparent. Certainly better if you like a constant opacity for all windows, instead of having to both define always on top and opacity in other open-source environments. You can also revert to the &quot;AeroShake&quot; functionality, whereby shaking the window minimizes all other windows. This is where criticism is needed because it could had easily been done so AeroSnap is default, but AquaSnap requiring a modifier like all the other Aqua features, which would avoid the need of having to choose from one or the other, since both behaviours have their advantages.<br><br><b>AquaGlass<\/b><br>An entirely frivolous feature but there if you need it; If you ever wanted window dragging to be like file dragging, where you can see <i>underneath<\/i> what you're dragging, then this is all the AquaGlass feature does.<br><br><b>Other stuff<\/b><br>If you wanted to go balls to the wall in productivity, and have a multi-function mouse or like to use the keyboard a lot more than your average person, then you can configure both mouse and keyboard to act exactly how you want, and how you perform most quickly. This means you can ditch geatures entirely and more-or-less replicate any open-source desktop of your choosing.<br><br>Features can be customized rather extensively, which means window manipulation works exactly the way you want it, and no more. Or maybe you want more, options are there for <i>lots<\/i> more as well. But the lack of a modifier for AquaShake is a bit of a bad mark when all other Aqua features are enabled the same way, and the explanations of these features in the AquaSnap UI can at worse be so vague that you need to try it in order to see it.<br><br>But overall it works, and collectively, it all comes together in a way that will aid productivity for people who remember they have it, and can remember how they have it set up. So the question remains; <i>Do you need all these features?<\/i> The answer to that is still no, because you can get along with the default features of Windows. But these are all features you might <i>want<\/i>, and if you think fifteen dollars is worth it for all the extra stuff you can do with your windows (as well, fix how some programs handle snapping), then it is well worth the purchase.<br><br>If you don't think it is, just wait for a sale and grab it then, or you can try out a majority of these features in the free version available from Nurgo
