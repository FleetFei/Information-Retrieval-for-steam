2\t\t\t\t\t\t\t\t\t\t<\/div>\r\n\r\n\t\t\t\t\t\r\n\r\n\r\n\r\n\r\n\r\n\r\n\t\t\t\t<div class=\"content\">\r\n\t\t\t\t\tI'll be updating this review later on as I develop my game to give more information as what this engine is truly capable of is somewhat unknown due to it being so new. I've found that working with firefly has been fantastic, Its possible to do games as simple or as complex as the user dictates, if you're a little intimidated by doing 3D for the first time you can still use your existing Fusion knoweledge by associating 2D objects to 3D graphics, this works fine.<br><br>On the more complex level there are some excelent features in the engine, I've found that the engine has a lot of inbuilt movement systems which is great, there's a system for doing FPS movement (though obviously you're going to want to use the examples in the workshop as they're REALLY good), theres a system for flight sim controls (Yaw, Pitch, Roll). There's also systems for controlling individual joints on a mesh though I've not explored these too much as one of the negatives is that the engine supports outdated model formats for animated meshes only- it would've been nicer if the engine supported FBX for Animated Meshes as those are pretty much universial and would allow us to export directly out of Maya, Max and Blender without any sort of conversion issues. The main problems I've had getting animations to work is how convoluted the export conversion process is- this isn't the fault of Firefly though, rather its a problem that Firefly will need to address in the future for the sake of ease of use.<br><br>As people have noted there is a bit of a lack of good documentation, not ever feature is documented and so it requires a bit of poking around to find out what things do, however a simple post of the forums usually gets an answer pretty quickly. I must say Clickteam have done really well with customer support on this product.<br><br>In terms of visual output the engine whilst dated, can output a lot of power for a 32 Bit Game engine (this engine isn't 64bit and as such you will need to ensure your artwork is well optimised for the engine- though you should be doing that anyway).<br><br>I think this is by far one of the most easier to use game engines out there. However it doesn't come with art pipeline tools as you'll need to use external programs for this. With that said you can still use billboards to create 3d games using sprites.<br><br>There has been a bit of negativity over the price, frankly the price is justifyable, 3D software is naturally much more expensive than 2D software, and if your intention is to go commercial with your software\/games then you'll make your investment back in no time anyway, there are far more expensive 3d software packages out there. Compared to other products on the market, whilst they may offer generous GPL licenses to their engines, the development time for games made in those will overtime turn out much longer than those created in Firefly. I've had this only for a month and I've already gotten a considerably large amount of progress on my games, whereas before I was using competing engines which were taking a long time to work with.<br><br>Clickteam Firefly is an investment for those wanting to make 3D games as a solo developer.\t\t\t\t\t<div class=\"gradient\"><\/div>\r\n\t\t\t\t<\/div>\r\n\r\n\t\t\t\t<div
