Anyway, back to profiles. Creating profiles for other programs is simple. Just below the Desktop profile settings there\u2019s a section called \u2018Process Bindings\u2019. The easiest way to get started is to run the program you want a profile for and then select it from the recently used section of the \u2018add a new process profile\u2019 menu. Simple. Once you\u2019ve done that you\u2019ve got the same options as the Desktop profile as well as Mouse settings for the specific profile and general profile settings, e.g. the name of the profile. The next time you start the program you\u2019ve just created a profile for, Controller Companion will automatically switch to the bindings for said program.<br><br>Another great thing about Controller Companion is that it supports the Steam Workshop. What does this mean for using the application? Well once you\u2019ve created a profile for a program you can upload it to the workshop (or update an existing workshop item) by exporting the profile. Once it\u2019s up other people can download the profile so they don\u2019t have to create their own for said program. <br><br>One last thing to mention is that whenever you launch an application with controller support, such as Steam Big Picture or a game with controller support, the application automatically disables itself to avoid any input duplication. It also does this for fullscreen programs such as other games though there do seem to be some exceptions to this.<br><br><div class=\"bb_h1\">Verdict <\/div><br><b>Essential Purchase <\/b><br><br>I\u2019ve been using this application with Netflix as well as a couple of games without controller support and it\u2019s been great.
